Error: "Import 'server' could not be resolved"

Location: backend/microservices/start_services.py (lines 30-33)

Root Cause:
- Used relative import from dynamic sys.path manipulation
- Import statement "from server import serve" couldn't resolve 'server' module
- sys.path.insert() approach doesn't work reliably with IDE linting/type checking

SOLUTION:
─────────
Replaced dynamic sys.path manipulation with importlib.util for explicit module loading:

OLD (Lines 24-35):
```python
async def start_route_service():
    try:
        logger.info("Starting Route Service...")
        route_service_path = Path(__file__).parent / 'route-service' / 'src'
        sys.path.insert(0, str(route_service_path))
        
        from server import serve as route_serve
        await route_serve()
```

NEW (Lines 24-41):
```python
async def start_route_service():
    try:
        logger.info("Starting Route Service...")
        route_service_path = Path(__file__).parent / 'route-service' / 'src'
        server_module_path = route_service_path / 'server.py'
        
        # Load server module dynamically
        spec = importlib.util.spec_from_file_location("route_server", server_module_path)
        route_server = importlib.util.module_from_spec(spec)
        sys.modules["route_server"] = route_server
        spec.loader.exec_module(route_server)
        
        await route_server.serve()
```

# UNIVERSAL ANALYSIS FRAMEWORK

Whenever you analyze or modify any part of the system, evaluate:

## 1. Functional Correctness

* Does it work logically?
* Are outputs accurate?
* Are edge cases handled?
* Are validations correct?

## 2. Data Integrity

* Database consistency
* Correct relationships
* Missing fields
* Incorrect mappings
* Data normalization issues

## 3. Performance

* Response latency
* Query efficiency
* API call frequency
* Caching effectiveness
* Memory usage

## 4. User Experience Speed

* Time to complete tasks
* Typing effort reduction
* Automation level
* Cognitive load
* Interaction smoothness

## 5. Error Handling & Recovery

* Network failures
* API downtime
* Invalid input
* Partial failures
* Retry mechanisms
* Graceful degradation

## 6. API Optimization

* Avoid unnecessary calls
* Cache intelligently
* Fallback strategies
* Rate limit protection
* Budget control

## 7. System Intelligence

* Predictive suggestions
* Autocomplete accuracy
* Smart defaults
* Context awareness
* AI intent detection

## 8. Security & Safety

* Payment integrity
* Authentication validation
* Injection risks
* Data exposure risks
* Authorization correctness

## 9. Integration Consistency

* Frontend ↔ Backend alignment
* Schema compatibility
* API contracts honored
* State synchronization

## 10. Scalability

* Multiple users
* Concurrent requests
* Queue handling
* Load spikes
* Future expansion readiness

## The Three Tables Explained

### 1️⃣ SQLite: `train_routes` (Railway Master Data)

**What it stores:**
```sql
CREATE TABLE train_routes (
  id INTEGER PRIMARY KEY,
  train_id INTEGER,                    -- Which train
  source_station_code VARCHAR,         -- Starting station code
  destination_station_code VARCHAR,    -- Ending station code
  route_sequence INTEGER,              -- Order in multi-leg journey
  distance_km FLOAT,
  estimated_duration FLOAT             -- Rough duration
);
```

**Example rows:**
```
Train 12301 (Rajdhani):
  Route 1: NDLS → NZM    (distance: 10 km, ~20 min)
  Route 2: NZM  → ALD    (distance: 200 km, ~3.5 hrs)
  Route 3: ALD  → MGS    (distance: 150 km, ~2.5 hrs)
  Route 4: MGS  → BPU    (distance: 250 km, ~5 hrs)
  Route 5: BPU  → BSB    (distance: 180 km, ~3 hrs)
  ...
  Final: BSB → KOTA      (distance: 100 km, ~2 hrs)
```

**Purpose**: Defines the "what" — which trains go where

### 2️⃣ PostgreSQL: `segments` (Travel Legs)

```sql
CREATE TABLE segments (
  id UUID PRIMARY KEY,
```

## Current Architecture Issues 🔧

### Issue #1: Data Island (The Main Problem)

```
SQLite has TRAINS but PostgreSQL doesn't know about them!

SQLite: "We have 40,000 train services"
PostgreSQL: *crickets*  →  "segments is empty"
RouteEngine: *screaming*  →  "Can't find routes!"
```

**Solution**: Build ETL script to read from SQLite, write to PostgreSQL

### Issue #2: No Sync Mechanism

```
SQLite updates daily with new schedules
PostgreSQL never gets updated
= STALE DATA in app
```

**Solution**: Schedule daily ETL runs (Cloud Scheduler)

```
Current: Developers manually add test segments
Better: Auto-sync from railway_manager.db

Current: seed_stations.py loads from JSON
```

## The Missing ETL Script

This is what needs to exist:

```python
# File: backend/etl/sqlite_to_postgres.py

def run_etl():
    """
    1. Open railway_manager.db (SQLite)
    2. Read train_schedule + train_routes + train_fares
    3. For EACH train service:
       - Get source & dest station IDs
       - Parse departure/arrival times
       - Calculate duration
       - Get cost
       - Get operating days
       → CREATE Segment in PostgreSQL
    4. Repeat for all ~40,000 train services
    5. Program finishes: segments table is populated
    
    Result: RouteEngine works! Users can search!
    """
```
